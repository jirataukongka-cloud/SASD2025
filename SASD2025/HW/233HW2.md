**สรุป Code Smells ตามหัวข้อ "Lazy Class" 4 ประเด็นหลัก**
**1.ความหมายของ Lazy Class**
<p>Lazy Class คือคลาสที่ถูกสร้างขึ้นมา แต่แทบไม่ได้ทำอะไรสำคัญ
ส่วนมากจะมี Method น้อยมาก หรือทำงานเล็ก ๆ ที่จริง ๆ แล้วสามารถเอาไปรวมกับคลาสอื่นได้
จากที่เรียนมา Lazy Class มักเกิดจากการออกแบบเผื่อไว้ก่อน
แต่สุดท้ายไม่ได้ใช้ตามที่คิด ทำให้มีคลาสเพิ่มขึ้นมาโดยไม่จำเป็น
ถึงโปรแกรมจะยังทำงานได้ปกติ แต่โค้ดจะดูรก และทำให้คนอ่านสับสน
ถ้าเจอ Lazy Class ส่วนใหญ่จะแก้โดยการรวมคลาสนั้นเข้ากับคลาสที่เรียกใช้งาน
เพื่อให้โค้ดสั้นลงและเข้าใจง่ายขึ้น</p>

**2.ตัวอย่าง Code ที่เกิดจาก Code Smells Lazy Class**
<p>
  public class DiscountCalculator {
    public double calculateDiscount(double price) {
        return price * 0.1;
    }
}
</p>

**กรณีถูกเรียกใช้ในคราสอื่น**
<p>
  public class OrderService {
    public double getFinalPrice(double price) {
        DiscountCalculator calculator = new DiscountCalculator();
        return price - calculator.calculateDiscount(price);
    }
}
</p>

**3.ตัวอย่างโค้ดหลังจาก Refactoring**
<p>
  public class OrderService {
    public double getFinalPrice(double price) {
        double discount = price * 0.1;
        return price - discount;
    }
}
</p>

**4.กรณีที่ Lazy Class ไม่ได้บ่งบอกว่าโค้ดมีปัญหา**
<p>ถึง Lazy Class จะเป็น code smell แต่ก็ไม่ได้แปลว่าต้องแก้ทุกครั้ง
บางคลาสถึงจะดูเล็ก แต่ก็ยังมีเหตุผลที่ควรเก็บไว้
อย่างแรกคือ คลาสนั้นอาจทำไว้เผื่ออนาคต
ตอนนี้โค้ดอาจยังไม่เยอะ แต่ต่อไปอาจมีการเพิ่มเงื่อนไขหรือการทำงานอื่น ๆ เข้ามา
อีกกรณีคือ คลาสช่วยให้โค้ดอ่านง่ายขึ้น
ถึงจะทำงานไม่มาก แต่ชื่อคลาสบอกชัดว่าเอาไว้ทำอะไร ทำให้คนอ่านเข้าใจง่าย
บางครั้งก็เป็นการออกแบบให้แต่ละคลาสทำหน้าที่เดียว
ถึงจะดูเล็ก แต่ช่วยแยกงานชัด และดูแลโค้ดง่ายขึ้น
สุดท้าย ถ้าคลาสนั้นเป็นส่วนหนึ่งของโครงสร้างระบบ
การเอาออกหรือรวมกับคลาสอื่นอาจกระทบส่วนอื่นได้ เลยไม่จำเป็นต้องแก้
</p>
